## WHERE WE ARE

Pipeline is end-to-end working on a real camera (Reolink P330).
Motion triggers → VLM analysis → SQLite record → dashboard live update.
Switched to Anthropic Claude Sonnet backend (llava:7b had GPU OOM with EasyOCR).
OCR reads text directly via Claude Vision — works on blurry fixed-lens footage.

## CAMERA

Reolink P330 at 10.0.0.58:554
- Connected via home network switch
- RTSP enabled via Reolink app
- Stream is H.265 (HEVC) — fixed-lens camera, no focus control whatsoever
- Fixed by replacing OpenCV VideoCapture with FFmpeg subprocess reader
  (core/camera.py pipes raw BGR frames from ffmpeg subprocess)
- RTSP URL: rtsp://admin:<password>@10.0.0.58:554/h264Preview_01_main
- NOTE: P330 has a fixed 2.8mm lens — no manual focus, no API/ONVIF control.
  For OCR/barcode use cases, camera must be mounted close (1–1.5m) to target.
  Varifocal alternative: RLC-811A, P430, P433.

## VLM BACKEND

Currently: Anthropic (claude-sonnet-4-6)
- Switched from llava:7b due to GPU OOM when EasyOCR ran alongside Ollama
- EasyOCR removed from OCR pipeline — Claude reads text directly from image
- pyzbar still used for barcode/QR decode (CPU, no VRAM needed)
- To switch back: VLM_BACKEND=local in .env
